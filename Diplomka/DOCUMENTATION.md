# Calorie Tracker Pro — Жобаның толық құжаттамасы

## 1. Жобаға шолу

**Calorie Tracker Pro** — калорияларды есепке алу, тамақтануды басқару және денсаулықты өмір салты үшін веб-қосымша. Жүйе тамақтану трекингі, өнімдер дүкені және жаттығулар каталогын бір платформада біріктіреді.

### 1.1 Атауы және мақсаты

| Параметр | Мәні |
|----------|------|
| **Атауы** | Calorie Tracker Pro |
| **Түрі** | Веб-қосымша (SPA + REST API) |
| **Мақсаты** | Калорияларды есепке алу, тамақтануды басқару, өнімдерді сатып алу, жаттығулар |
| **Мақсатты аудитория** | Тамақтану мен денсаулыққа қамқорлық жасайтын пайдаланушылар |

## 2. Жобаның құндылықтары (презентация үшін)

### 2.1 Пайдаланушы құндылықтары

| Құндылық | Сипаттама |
|----------|-----------|
| **Ыңғайлылық** | Тамақтануды есепке алу, сатып алу және жаттығулар үшін бір платформа — бәрі бір жерде |
| **Бақылау** | Жеке норманы ескере отырып, калориялар мен БЖУ-ді нақты уақытта қадағалау |
| **Жекелендіру** | Профильді баптау: жыныс, жасы, салмағы, бойы, белсенділік деңгейі, калория нормы және макростар |

| **Уақытты үнемдеу** | Тамақтану туралы жазбаларды жасау және каталогтан өнімдерді сатып алу |
| **Мотивация** | Дайын бағдарламалар мен жаттығулармен жаттығулар бөлімі |

| Құндылық | Сипаттама |

| **Әкімшілендіру** | Администратор панелі: пайдаланушылар, тауарлар, тапсырыстар |

## 4. Архитектура

### 4.1 Жоба құрылымы

```
Diplomka/
├── Backend/                 # Node.js API
│   ├── src/
│   │   ├── config/          # БД конфигурациясы
│   │   ├── controllers/     # Контроллерлер
│   │   ├── middleware/      # auth, validation, upload
│   │   ├── routes/          # API маршруттары
│   │   └── index.js         # Кіру нүктесі
│   ├── database/

│   │   └── init.sql         # Бастапқы схема
│   ├── uploads/             # Жүктелген өңдеулер
│   ├── docker-compose.yml
│   └── package.json
│
└── Front end/               # React SPA
    ├── src/
    │   ├── api/             # API клиенттері
    │   ├── components/      # Header, ProtectedRoute, AdminRoute
    │   ├── data/            # Статикалық деректер (жаттығулар)
    │   ├── pages/           # Қосымша беттері
    │   └── App.jsx
    └── package.json
```

### 4.2 Әрекеттесу схемасы

```
[Пайдаланушы] → [React SPA] → [REST API] → [PostgreSQL]
                     ↑
              JWT localStorage-да
```

## 5. Функционалдылық

### 5.1 Пайдаланушы рөлдері

| Рөл | Қол жетімділік |
|-----|----------------|
| **user** | Басты бет, Профиль, Дүкен, Себет, Жаттығулар |
| **admin** | Жоғарыдағылар + Тауарлар, Тапсырыстар, Пайдаланушылар |

### 5.2 Беттер мен функциялар

| Бет | Маршрут | Сипаттама |
|-----|---------|-----------|
| Басты бет | `/` | Тамақтану туралы жазбаларды қосу, жазбалар тізімі, статистика (калориялар, БЖУ) |
| Профиль | `/profile` | Баптаулар: жыныс, жасы, салмағы, бойы, белсенділік, нормалар, парольді өзгерту |
| Дүкен | `/shop` | Бағасы, калориялары, БЖУ бар өнімдер каталогы; себетке қосу |
| Себет | `/cart` | Себетті қарау, көлемді өзгерту, тапсырыс рәсімдеу (тесті төлем) |
| Жаттығулар | `/workouts` | Сипаттама және жаттығулармен жаттығулар каталогы |
| Тауарлар (admin) | `/products` | Өңдеулері бар тауарларды CRUD |
| Тапсырыстар (admin) | `/orders` | Тапсырыстар тізімі |
| Пайдаланушылар (admin) | `/users` | Пайдаланушыларды басқару |
| Кіру | `/login` | Авторизация |
| Тіркелу | `/registration` | Жаңа пайдаланушыны тіркеу |

### 5.3 Негізгі мүмкіндіктер

- **Тамақтануды есепке алу**: күні, өнім, калориялар, БЖУ бар жазбаларды қосу
- **Жазбаларды сүзгілеу**: күн бойынша (бастап/дейін)
- **Статистика**: бүгін, норма, қалды, барлығы; макростар (ақуыздар, майлар, көмірсулар)
- **Профиль**: жеке нормаларды есептеу және сақтау
- **Дүкен**: тауарларды қарау, себетке қосу
- **Себет**: көлемді өзгерту, жою, мекенжай және тесті картамен checkout
- **Жаттығулар**: егжей-тегжейлі сипаттама және жаттығулармен статикалық каталог

## 6. API

### 6.1 Негізгі URL

- Локальді: `http://localhost:3001/api`
- Health: `GET /api/health`

### 6.2 Эндпоинттер

| Әдіс | Жол | Сипаттама | Auth |
|------|-----|-----------|------|
| POST | `/auth/register` | Тіркелу | — |
| POST | `/auth/login` | Кіру | — |
| GET | `/auth/profile` | Пайдаланушы профилі | ✓ |
| PATCH | `/auth/profile` | Профильді жаңарту | ✓ |
| GET | `/entries` | Жазбалар тізімі (dateFrom, dateTo) | ✓ |
| POST | `/entries` | Жазба құру | ✓ |
| DELETE | `/entries/:id` | Жазбаны жою | ✓ |
| GET | `/entries/stats` | Калориялар статистикасы | ✓ |
| GET | `/products` | Жария тауарлар каталогы | ✓ |
| GET | `/cart` | Себет | ✓ |
| POST | `/cart` | Себетке қосу (productId, quantity) | ✓ |
| PATCH | `/cart/:id` | Көлемді өзгерту | ✓ |
| DELETE | `/cart/:id` | Себеттен жою | ✓ |
| POST | `/cart/checkout` | Тапсырыс рәсімдеу (cardNumber, address) | ✓ |
| CRUD | `/admin/users/*` | Пайдаланушыларды басқару | Admin |
| CRUD | `/admin/products/*` | Тауарларды басқару | Admin |
| GET | `/admin/orders/*` | Тапсырыстар | Admin |

### 6.3 Аутентификация

- JWT тақырыпта: `Authorization: Bearer <token>`
- Токен клиентте `localStorage`-да сақталады

## 7. Деректер базасы

### 7.1 Негізгі кестелер

| Кесте | Мақсаты |
|-------|---------|
| `users` | Пайдаланушылар (аты, email, пароль, профиль, рөл, is_banned) |
| `food_entries` | Тамақтану туралы жазбалар (user_id, product_name, calories, protein, fat, carbs, entry_date) |

| `cart_items` | Себет (user_id, product_id, quantity) |

| `order_items` | Тапсырыс позициялары (order_id, product_id, product_name, quantity, price) |

### 7.2 Миграциялар

- `init.sql` — users
- `002_profile.sql` — профиль өрістері
- `003_food_entries.sql` — тамақтану туралы жазбалар
- `004_add_role.sql` — пайдаланушы рөлі
- `005_optional_set_admin.sql` — admin тағайындау мүмкіндігі
- `006_add_is_banned.sql` — пайдаланушыларды байлау
- `007_create_products.sql` — тауарлар
- `008_add_product_image.sql` — тауар өңдеулері
- `009_add_product_price.sql` — тауар бағасы
- `010_create_cart.sql` — себет
- `011_create_orders.sql` — тапсырыстар
- `012_add_order_address.sql` — жеткізу мекенжайы

## 8. Орнату

### 8.1 Локальді іске қосу (Docker-сыз)

1. **PostgreSQL**: орнатып, `calorie_tracker` БД құру
2. **Backend**:
   ```bash
   cd Backend
   cp .env.example .env   # DATABASE_URL, JWT_SECRET баптау
   npm install
   npm run dev
   ```
3. **Frontend**:
   ```bash
   cd "Front end"
   npm install
   npm run dev
   ```
4. `http://localhost:5173` немесе `http://localhost:5174` ашу

### 8.2 Docker-мен іске қосу

```bash
cd Backend
docker-compose up -d
```

- Backend: `http://localhost:3001`
- PostgreSQL: `localhost:5432`

Frontend бөлек іске қосылады: `cd "Front end" && npm run dev`

### 8.3 Орта айнымалылары (.env)

| Айнымалы | Сипаттама |
|----------|-----------|
| PORT | Сервер порты (әдепкі 3001) |
| DATABASE_URL | PostgreSQL қосылу жолы |
| JWT_SECRET | JWT қолтаңбасы үшін құпия |
| FRONTEND_URL | CORS үшін рұқсат етілген origins (үтірімен) |

## 10. Презентация слайдтары (қысқа жоспар)

1. **Титулдық слайд**: Calorie Tracker Pro — калорияларды есепке алу және денсаулықты өмір салты
2. **Мәселе**: Тамақтануды басқару және пайдалы өнімдерді бір жерде сатып алу қиын
3. **Шешім**: Бір платформа: трекинг + дүкен + жаттығулар
4. **Құндылықтар**: Ыңғайлылық, бақылау, жекелендіру, мотивация
5. **Технологиялар**: React, Node.js, PostgreSQL, Docker
6. **Архитектура**: Frontend — API — БД схемасы
7. **Функциялар**: Басты бет, дүкен, себет, профиль скриншоттары
8. **Рөлдер**: Пайдаланушы vs Администратор
9. **Қауіпсіздік**: JWT, bcrypt, валидация
10. **Орнату**: Docker Compose, орта айнымалылары
11. **Қорытынды**: Толық функционалды қосымша, кеңейтуге дайын
